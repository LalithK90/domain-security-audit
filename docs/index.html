Domain Security Audit Platform

A research-grade platform for large-scale domain infrastructure mapping and security posture assessment. Built for security researchers, domain administrators, and academic institutions worldwide.

View GitHub Repository Get Started

Key Capabilities

Subdomain Enumeration

12 discovery methods including Certificate Transparency, public databases, DNS analysis, and seed lists.

Security Assessments

30+ comprehensive checks across HTTPS/TLS, email security, DNS security, and HTTP headers.

Research-Grade Outputs

CSV, Excel, JSON, and Markdown reports with statistics and visualizations.

Ethical by Design

Passive-only mode uses public data sources, enabling legally compliant research.

Two Usage Models

Model 1 — Academic Research

Passive-only mode for legally compliant research and publication worldwide.

  • Certificate Transparency, public databases, DNS, WHOIS
  • No permission required (global compliance)
  • Study any domain without authorization

Model 2 — Infrastructure Audit

Active security assessment for your own or authorized infrastructure.

  • HTTPS/TLS validation, email security, headers
  • Requires ownership or written authorization
  • Deep security posture reporting

Platform Capabilities & Standards

Production-ready security assessment framework. 30+ security checks aligned with NIST, OWASP, and CIS standards. Comprehensive passive enumeration and optional active scanning with standards-validated methodology.

Quick Start

  1. Install dependencies: Python 3.8+
  2. Install packages: pip install -r src/requirements.txt
  3. Create .env: DOMAIN=example.com and ALLOW_ACTIVE_PROBES=false
  4. Run scan: ./run.sh or python src/app.py
  5. Review outputs in out/ folder

Legal & Ethical Use

Passive-Only (Recommended & Global)

Uses only public data sources. Legal in all jurisdictions worldwide.

  • Certificate Transparency logs
  • Public vulnerability databases
  • DNS and WHOIS records
  • No authorization needed

Active Scanning (Authorization Required)

For domains you own or have written permission to test.

  • HTTP/TLS probing
  • Email server checks
  • Security header validation
  • Requires ownership or permission
